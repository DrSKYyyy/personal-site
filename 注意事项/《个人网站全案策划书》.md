## 一、 项目综述与核心定位

### 1.1 项目背景

我希望建立一个**非功利性、高度个人化**的网站。它不仅是简历的延伸，更是您主体性的延展。网站需要承载**社会性展示**（介绍自我）与**主体性生长**（通过持续记录见证成长）双重使命。

### 1.2 核心定位关键词

- **数字花园**：非流水账博客，而是精心打理、随时间生长的思想空间。
- **追风少年**：赤诚、爽朗、不畏风、有力量且平易近人。
- **长期主义**：架构设计以 5-10 年为周期，确保内容可迁移、可扩展。
- **AI 原生**：借助 AI 实现想法，关注逻辑理解而非代码细节。

### 1.3 成功度量指标

- **访客侧**：访问 30 秒内能感知到“这是一个有趣的、真诚的人”。
- **自我侧**：在 Obsidian 写完 Markdown 后，**5 分钟内**可完成网站发布。
- **体验侧**：PC 端 Lighthouse 性能评分 > 90 分。

## 二、 网站人格画像与视觉语言

### 2.1 身份标识——“少年”

- **视觉图腾**：卡通风格的大头照（带灿烂笑容），配以动态“嘻嘻”文字或轻快音效。
- **氛围关键词**：阳光、风、手绘感、通透留白、偶尔搞怪。
- **色彩基调**：
  - **主色**：**天空蓝**（`#5DADE2`）与 **暖阳黄**（`#F4D03F`）的渐变，象征希望与快乐。
  - **背景**：**米白/极浅灰**（`#F8F9FA`），保持干净清爽的阅读感。
  - **强调色**：**活力橙**（`#E67E22`），用于按钮、链接悬停，增加跳跃感。
- **字体系统**：
  - **标题/首屏**：手写风格字体（如 **Gaegu**、**ZCOOL KuaiLe**），体现少年感。
  - **正文**：高可读性无衬线字体（**Inter** 或 **System UI**）。

### 2.2 首屏方案（第一印象锚点）

- **布局**：居中大标题 `Hi, I'm [您的名字]` + 右侧（或下方）手绘风格卡通大头照（带 GIF 眨眼/微笑效果）。
- **交互**：鼠标悬停大头照时，弹出随机台词气泡（“今天很开心！”、“又学到新东西啦！”）。
- **背景**：极简动态背景，如缓慢飘动的**云朵线条画**或**风吹动的粒子效果**。

## 四、 核心技术架构与选型逻辑

> **设计原则**：用 **JAMstack（JavaScript, APIs, Markup）** 架构，平衡动态功能与静态托管成本。

### 4.1 推荐技术栈：Astro + GitHub Pages

| 层级       | 选用技术                            | **为什么这样选？（逻辑解释）**                                                                                                                                                               |
| :------- | :------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **框架**   | **Astro**                       | 1. **内容为王**：原生支持 Markdown/MDX，天生为博客而生。2. **零 JS 默认**：生成的页面是纯 HTML，**加载极快**，满足“丝滑”需求。3. **岛屿架构**：只在需要交互的地方（如评论框、筛选器）加载 JS，性能优化到极致。4. **易于扩展**：未来想加入 React/Vue 组件（如地图、复杂动效）可直接集成。 |
| **部署**   | **GitHub Pages**                | 免费、稳定，与 Git 工作流完美结合，每次推送代码自动部署。                                                                                                                                                 |
| **内容管理** | **Git-based (Decap CMS)**       | **解决您的痛点**：提供一个**网页版后台**让您直接在浏览器里写 Markdown 博客，并自动提交到 GitHub。无需打开 IDE 改代码。且支持 **Obsidian 双向联动**（见第五章）。                                                                          |
| **数据统计** | **Cloudflare Web Analytics**    | 隐私优先、不收集个人数据、不影响页面性能评分，且能统计到“时段”流量。                                                                                                                                             |
| **图片处理** | **Astro 图片组件 + Cloudflare CDN** | 自动生成缩略图和多尺寸格式（WebP/AVIF）。点击放大使用 `photoswipe` 库实现画廊效果。                                                                                                                           |

## 五、 核心工作流：与 Obsidian 的“数字永生”联动

这是您最关心的**可扩展性与内容安全**的保障。

### 5.1 文件组织结构

```
my-website/
├── src/
│   ├── content/
│   │   ├── blog/
│   │   │   ├── 2024-04-20-my-first-post.md
│   │   │   └── 2025-01-01-new-year.md
│   │   ├── projects/
│   │   │   └── project-a.md
│   │   └── special/
│   │       └── books.json (或 mdx)
```

### 5.2 Obsidian 工作流设计

1. **本地写作**：您电脑上有一个 `Obsidian Vault`（本地文件夹）。
2. **软链接/同步**：将 `Obsidian Vault` 下的 `blog` 文件夹**软链接**或**同步**到上述 `src/content/blog/` 目录。
3. **一键发布**：
   - **方式 A（极客流）**：在 Obsidian 写完后，打开终端，输入 `git add . && git commit -m "新文章" && git push`。（AI 可帮您写一个一键脚本）
   - **方式 B（后台流）**：访问 `yoursite.com/admin`（Decap CMS 后台），同样可以编辑 Markdown，保存即自动 Push 到 GitHub。
4. **导入导出**：所有文章就是 `.md` 文件，**天然支持任意迁移**。即使未来不用 Astro 了，您的文字资产毫发无损。

### 5.3 高级时间归档实现

- 利用 Astro 的 `getCollection` API，在构建时读取所有 Markdown 文件的 `frontmatter` 中的 `date` 字段。
- 在前端实现一个下拉菜单，按 **年/月/日** 和 **时段（凌晨/早晨/上午/中午/下午/晚上/深夜）** 动态过滤列表。*（此项功能 AI 生成 React/Vue 组件即可实现）*
