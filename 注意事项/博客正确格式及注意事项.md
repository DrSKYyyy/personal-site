# 从 Obsidian 复制到博客的完整指南

## 一、Frontmatter（文章头部信息）

### 正确模板

```yaml
---
title: 文章标题          # 必填
date: "2026-05-01"      # 必填，日期必须加引号
time: "14:30"            # 可选，用于文章列表按时段分类（时:分）
tags:                    # 可选
  - 标签1
  - 标签2
visibility: 公开         # 可选，默认"公开"。可选值：公开 / 私密 / 草稿
description: 摘要文字   # 可选，文章列表里会显示
---
```

### 常见错误

| 错误写法 | 问题 | 正确写法 |
|---------|------|---------|
| `date: 2026-05-01` | YAML 自动解析成日期类型，schema 报错 | `date: "2026-05-01"` |
| `title：文章标题` | 全角冒号，YAML 不识别 | `title: 文章标题` |
| `***` 开头 | 不是 frontmatter 分隔符 | `---`（三个减号） |
| `----------------` 结尾 | 不是 frontmatter 分隔符 | `---`（三个减号） |
| 缺少 `title` 字段 | schema 要求必填 | 加上 `title: 标题` |

### 分隔符示意图

```markdown
---                    ← 三个减号开头
title: 标题
date: "2026-05-01"
---
                      ← 这里空一行
正文从这里开始...
```

## 二、标题级别（`#` 的数量）

### 文章内的章节标题请使用 `##`（二级标题）

```markdown
## 为什么要建立这个网站

正文内容...

## 网站开发顺利吗？

正文内容...
```

**为什么？** 网站 CSS 只给 `##`（h2）自动添加了美观的分割线：
```
────────────────────────────────────  ← 半透明灰色线，自动生成
```

如果用了 `#`（一级标题），不会有分割线。

### 各标题级别效果

| Markdown | 效果 |
|----------|------|
| `# 标题` | 无分割线 |
| `## 标题` | ✅ 有分割线 |
| `### 标题` | 无分割线 |

> 不要在文章里用手动画线 `---` 或 `***` 来分割，很不美观。用 `##` 自然就有线了。

## 三、段落缩进

### 不要在行首敲空格

```markdown
   这行前面有4个空格    ← ❌ 会被 Markdown 当成代码块
```

行首的 4 个空格或 1 个 Tab 会被渲染成代码块（等宽字体 + 灰色背景框），而不是缩进。

### 需要缩进时用 `&emsp;`

在段落开头加 `&emsp;`（一个 = 一个汉字宽度）：

```markdown
&emsp;&emsp;这是首行缩进了两个字的段落。
```

一个 `&emsp;` = 一个汉字的宽度，所以缩进两格就加两个。

## 四、空行与间距

标题和正文之间的间距由 CSS 自动控制，**不需要也不应该多敲空行**。正常写 Markdown 就行：

```markdown
## 标题

正文...               ← 空一行是正常的
```

多敲的空行在渲染时不会产生额外间距。

## 五、文件命名

- 文件名就是文章在网址上的 id
- 中文文件名可以用（但建议保持唯一）
- 不要和其他文件重名（含中文转义后冲突）
- 示例：`2026-05-01-hello-world.md`

---

## 六、添加照片

照片放在 `public/images/` 文件夹里，然后在 frontmatter 或正文中引用：

```markdown
![描述文字](/images/文件名.jpg)
```

---

## 七、速查清单

每次从 Obsidian 复制文章到 `src/content/blog/` 后，检查以下 6 点：

- [ ] frontmatter 分隔符是 `---`（不是 `***` 或别的）
- [ ] `date: "2026-05-01"`（加了引号）
- [ ] 所有冒号都是半角 `:`（不是全角 `：`）
- [ ] 章节标题用了 `##`（不是 `#`）
- [ ] 没有行首的空格或 Tab
- [ ] 如果需要时段分类，加了 `time: "14:30"`

确保这 6 点就没问题了。

## 八、时段分类说明

在文章列表里有一个"时段"筛选功能（凌晨/早晨/上午/中午/下午/晚上/深夜）。

如需使用，在 frontmatter 中添加 `time` 字段：

```yaml
time: "14:30"   # 格式：时:分，24小时制
```

各时段对应的小时范围：

| 时段 | 小时范围 |
|------|---------|
| 凌晨 | 0:00-4:59 |
| 早晨 | 5:00-6:59 |
| 上午 | 7:00-10:59 |
| 中午 | 11:00-12:59 |
| 下午 | 13:00-17:59 |
| 晚上 | 18:00-21:59 |
| 深夜 | 22:00-23:59 |

不填 `time` 字段则不会显示时段标签。
