从Obsidian复制到博客的注意事项

一、Frontmatter（文章头部信息）

正确模板

---
title: 文章标题          # 必填
date: "2026-05-01"      # 必填，日期必须加引号
time: "14:30"            # 可选，用于文章列表按时段分类（时:分）
pinned: 1                # 可选，数字越小越靠前，有该字段的文章会出现在"置顶文章"区域
tags:                    # 可选
  - 标签1
  - 标签2
visibility: 公开         # 可选，默认"公开"。可选值：公开 / 私密 / 草稿
description: 摘要文字   # 可选，文章列表里会显示
---

常见错误

错误写法问题正确写法
date: 2026-05-01YAML 自动解析成日期类型，schema 报错date: "2026-05-01"
title：文章标题全角冒号，YAML 不识别title: 文章标题
*** 开头不是 frontmatter 分隔符---（三个减号）
---------------- 结尾不是 frontmatter 分隔符---（三个减号）
缺少 title 字段schema 要求必填加上 title: 标题

分隔符示意图

---                    ← 三个减号开头
title: 标题
date: "2026-05-01"
---
                      ← 这里空一行
正文从这里开始...

二、标题级别（# 的数量）

文章内的章节标题请使用 ##（二级标题）

## 为什么要建立这个网站

正文内容...

## 网站开发顺利吗？

正文内容...

为什么？ 网站 CSS 只给 ##（h2）自动添加了美观的分割线：

────────────────────────────────────  ← 半透明灰色线，自动生成

如果用了 #（一级标题），不会有分割线。

各标题级别效果

Markdown效果
# 标题无分割线
## 标题✅ 有分割线
### 标题无分割线

不要在文章里用手动画线 --- 或 *** 来分割，很不美观。用 ## 自然就有线了。

三、段落缩进

不要在行首敲空格

   这行前面有4个空格    ← ❌ 会被 Markdown 当成代码块

行首的 4 个空格或 1 个 Tab 会被渲染成代码块（等宽字体 + 灰色背景框），而不是缩进。

需要缩进时用 &emsp;

在段落开头加 &emsp;（一个 = 一个汉字宽度）：

&emsp;&emsp;这是首行缩进了两个字的段落。

一个 &emsp; = 一个汉字的宽度，所以缩进两格就加两个。

四、空行与间距

标题和正文之间的间距由 CSS 自动控制，不需要也不应该多敲空行。正常写 Markdown 就行：

## 标题

正文...               ← 空一行是正常的

多敲的空行在渲染时不会产生额外间距。

五、文件命名

  • 文件名就是文章在网址上的 id
  • 中文文件名可以用（但建议保持唯一）
  • 不要和其他文件重名（含中文转义后冲突）
  • 示例：2026-05-01-hello-world.md

六、添加照片

照片放在 public/images/ 文件夹里，然后在正文中引用：

Markdown 语法（不能调大小）：

![描述文字](/images/文件名.jpg)

HTML 语法（可以调大小）：

<img src="/images/文件名.jpg" alt="描述" width="400">
  • 路径 public/images/xxx.jpg → 写为 /images/xxx.jpg
  • 分隔符用 /（正斜杠），不要用 \
  • 只设 width 不设 height，图片自动等比缩放

图片后换行

<img> 是行内元素，多个图片默认并排。如需换行，在图片后面加 <br>：

<img src="/images/照片1.jpg" alt="" width="300"><br>
<img src="/images/照片2.jpg" alt="" width="300">
<br>
&emsp;&emsp;正文从这里开始...

七、注释语法

YAML（frontmatter）注释：使用 #

---
title: "标题"          # 行尾注释
# 整行注释
date: "2026-05-01"
# pinned: 1           # 注释掉就不置顶了
---

Markdown 正文注释：用 HTML 注释

<!-- 这行不会显示在页面上 -->

在 siteData.js 里：

// 单行注释
/* 多行注释 */

八、速查清单

每次从 Obsidian 复制文章到 src/content/blog/ 后，检查以下 7 点：

  • frontmatter 分隔符是 ---（不是 *** 或别的）
  • date: "2026-05-01"（加了引号）
  • 所有冒号都是半角 :（不是全角 ：）
  • 章节标题用了 ##（不是 #）
  • 没有行首的空格或 Tab
  • 如果需要时段分类，加了 time: "14:30"
  • 如果需要置顶，加了 pinned: 1

确保这 7 点就没问题了。

九、时段分类说明

在文章列表里有一个”时段”筛选功能（凌晨/早晨/上午/中午/下午/晚上/深夜）。

如需使用，在 frontmatter 中添加 time 字段：

time: "14:30"   # 格式：时:分，24小时制

各时段对应的小时范围：

时段小时范围
凌晨0:00-4:59
早晨5:00-6:59
上午7:00-10:59
中午11:00-12:59
下午13:00-17:59
晚上18:00-21:59
深夜22:00-23:59

不填 time 字段则不会显示时段标签。

十、置顶文章

如需让文章置顶，在 frontmatter 中添加 pinned 字段：

pinned: 1     # 数字越小越靠前

有 pinned 字段的文章会出现在文章列表最上方的”📌 置顶文章”区域，可按 pinned 的值调整顺序。如果多篇置顶：

pinned: 1    # 排第一
pinned: 2    # 排第二
pinned: 3    # 排第三

不填则不置顶，按正常时间排序。
