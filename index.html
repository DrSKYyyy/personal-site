`##`（二级标题）\n\n```markdown\n## 为什么要建立这个网站\n\n正文内容...\n\n## 网站开发顺利吗？\n\n正文内容...\n```\n\n**为什么？** 网站 CSS 只给 `##`（h2）自动添加了美观的分割线：\n```\n──────────────────────────────────── ← 半透明灰色线，自动生成\n```\n\n如果用了 `#`（一级标题），不会有分割线。\n\n### 各标题级别效果\n\n| Markdown | 效果 |\n|----------|------|\n| `# 标题` | 无分割线 |\n| `## 标题` | ✅ 有分割线 |\n| `### 标题` | 无分割线 |\n\n> 不要在文章里用手动画线 `---` 或 `***` 来分割，很不美观。用 `##` 自然就有线了。\n\n---\n\n## 三、段落缩进\n\n### 不要在行首敲空格\n\n```markdown\n 这行前面有4个空格 ← ❌ 会被 Markdown 当成代码块\n```\n\n行首的 4 个空格或 1 个 Tab 会被渲染成代码块（等宽字体 + 灰色背景框），而不是缩进。\n\n### 需要缩进时用 `&emsp;`\n\n在段落开头加 `&emsp;`（一个 = 一个汉字宽度）：\n\n```markdown\n&emsp;&emsp;这是首行缩进了两个字的段落。\n```\n\n一个 `&emsp;` = 一个汉字的宽度，所以缩进两格就加两个。\n\n---\n\n## 四、空行与间距\n\n标题和正文之间的间距由 CSS 自动控制，**不需要也不应该多敲空行**。正常写 Markdown 就行：\n\n```markdown\n## 标题\n\n正文... ← 空一行是正常的\n```\n\n多敲的空行在渲染时不会产生额外间距。\n\n---\n\n## 五、文件命名\n\n- 文件名就是文章在网址上的 id\n- 中文文件名可以用（但建议保持唯一）\n- 不要和其他文件重名（含中文转义后冲突）\n- 示例：`2026-05-01-hello-world.md`\n\n---\n\n## 六、添加照片\n\n照片放在 `public/images/` 文件夹里，然后在正文中引用：\n\n**Markdown 语法**（不能调大小）：\n\n```markdown\n![描述文字](/images/文件名.jpg)\n```\n\n**HTML 语法**（可以调大小）：\n\n```markdown\n<img src=\"/images/文件名.jpg\" alt=\"描述\" width=\"400\">\n```\n\n- 路径 `public/images/xxx.jpg` → 写为 `/images/xxx.jpg`\n- 分隔符用 `/`（正斜杠），不要用 `\\`\n- 只设 `width` 不设 `height`，图片自动等比缩放\n\n### 图片后换行\n\n`<img>` 是行内元素，多个图片默认并排。如需换行，在图片后面加 `<br>`：\n\n```markdown\n<img src=\"/images/照片1.jpg\" alt=\"\" width=\"300\"><br>\n<img src=\"/images/照片2.jpg\" alt=\"\" width=\"300\">\n<br>\n&emsp;&emsp;正文从这里开始...\n```\n\n---\n\n## 七、注释语法\n\n**YAML（frontmatter）注释**：使用 `#`\n\n```yaml\n---\ntitle: \"标题\" # 行尾注释\n# 整行注释\ndate: \"2026-05-01\"\n# pinned: 1 # 注释掉就不置顶了\n---\n```\n\n**Markdown 正文注释**：用 HTML 注释\n\n```markdown\n<!-- 这行不会显示在页面上 -->\n```\n\n**在 `siteData.js` 里**：\n\n```js\n// 单行注释\n/* 多行注释 */\n```\n\n---\n\n## 八、速查清单\n\n每次从 Obsidian 复制文章到 `src/content/blog/` 后，检查以下 7 点：\n\n- [ ] frontmatter 分隔符是 `---`（不是 `***` 或别的）\n- [ ] `date: \"2026-05-01\"`（加了引号）\n- [ ] 所有冒号都是半角 `:`（不是全角 `：`）\n- [ ] 章节标题用了 `##`（不是 `#`）\n- [ ] 没有行首的空格或 Tab\n- [ ] 如果需要时段分类，加了 `time: \"14:30\"`\n- [ ] 如果需要置顶，加了 `pinned: 1`\n\n确保这 7 点就没问题了。\n\n## 九、时段分类说明\n\n在文章列表里有一个\"时段\"筛选功能（凌晨/早晨/上午/中午/下午/晚上/深夜）。\n\n如需使用，在 frontmatter 中添加 `time` 字段：\n\n```yaml\ntime: \"14:30\" # 格式：时:分，24小时制\n```\n\n各时段对应的小时范围：\n\n| 时段 | 小时范围 字段则不会显示时段标签。\n\n## 十、置顶文章\n\n如需让文章置顶，在 frontmatter 中添加 `pinned` 字段：\n\n```yaml\npinned: 1 # 数字越小越靠前\n```\n\n有 `pinned` 字段的文章会出现在文章列表最上方的\"📌 置顶文章\"区域，可按 `pinned` 的值调整顺序。如果多篇置顶：\n\n```yaml\npinned: 1 #
